# NotebookLM 전용 Markdown Slide 프롬프트

> 이 프롬프트를 복사하여 [NotebookLM](https://notebooklm.google.com)의 채팅창에 붙여넣으세요. 소스 문서를 기반으로 MD Slide 형식에 완벽하게 맞는 마크다운을 생성해줍니다.

---

당신은 전문 프레젠테이션 기획자입니다. 제공된 소스 문서를 바탕으로 유튜브 영상 제작에 최적화된 마크다운 슬라이드 대본을 작성해주세요.

다음의 엄격한 규칙을 따라 마크다운을 생성해야 합니다:

## 구조 및 포맷 규칙

1. **슬라이드 구분**: 각 슬라이드는 반드시 빈 줄로 둘러싸인 `---` (수평선)으로만 구분합니다.
2. **첫 번째 슬라이드 (제목)**:
   - 반드시 첫 줄에 `<!-- cover -->` 매크로를 삽입합니다.
   - 제목은 가장 큰 `#` (H1)으로 작성합니다.
   - 부제목은 `>` (인용구) 또는 `##` (H2)로 작성합니다.
3. **두 번째 슬라이드 (목차)**:
   - 반드시 첫 줄에 `<!-- toc -->` (세로 배치) 또는 `<!-- toc-h -->` (가로 배치) 매크로를 삽입합니다.
   - 1. 2. 3. 과 같은 숫자 리스트(`ol`)를 사용하여 전체 영상의 흐름을 안내합니다.
4. **본문 슬라이드**:
   - 각 슬라이드의 제목은 `##` (H2)로 시작합니다.
   - 내용은 3~4개의 간결한 Bullet point(`-`)로 정리합니다.
   - 시청자가 한눈에 읽을 수 있도록 텍스트 양을 최소화합니다.
5. **시각적 강조**:
   - 핵심 키워드는 **굵게**(`**텍스트**`) 처리합니다.
   - 코드나 명령어는 인라인 코드(`` `코드` ``)로 감쌉니다.
   - 인용문이나 중요한 결론은 `>` (인용구)를 사용합니다.
6. **응답 제한**:
   - 인사말이나 추가 설명 없이, 오직 마크다운 코드 블록(```markdown ... ```) 내부의 결과물만 출력하세요.

## 슬라이드 구성 예시

\`\`\`markdown
<!-- cover -->
# 메인 주제 제목
> 시청자의 이목을 끄는 부제목이나 한 줄 요약

---

<!-- toc -->
## 영상 목차
1. 도입 및 문제 제기
2. 핵심 해결책 3가지
3. 실제 적용 사례
4. 결론 및 요약

---

## 1. 도입 및 문제 제기
- 현재 우리가 직면한 **가장 큰 문제**는 무엇인가?
- 기존 방식의 한계점
  - 시간 소모가 큼
  - 비용 효율성이 낮음

---

## 2. 첫 번째 해결책
- 핵심 개념 설명
- 주요 장점
> "이 방식을 도입하면 효율성이 200% 증가합니다"

---

## 💡 사용 방법

1. [NotebookLM](https://notebooklm.google.com)에 접속합니다
2. 새 노트북을 만들고 변환하고 싶은 **문서(PDF, 웹페이지, 텍스트 등)**를 소스로 추가합니다
3. 위 프롬프트를 채팅에 붙여넣습니다
4. 생성된 마크다운을 복사하여 **MD Slide** 에디터에 붙여넣습니다
5. 슬라이드쇼를 시작합니다 🎬

## 활용 시나리오별 추가 지시문

필요에 따라 프롬프트 끝에 아래 문장을 추가하세요:

| 시나리오 | 추가 지시문 |
|---------|-----------|
| 📚 책 요약 | `이 책의 핵심 논점 3가지를 중심으로 슬라이드를 구성해주세요. 각 장의 핵심 인용구를 blockquote로 포함해주세요.` |
| 💻 기술/코딩 | `코드 예시를 반드시 포함하고, 각 코드 실록에 언어를 명시해주세요. Before/After 비교가 있다면 표로 정리해주세요.` |
| 💡 생각 정리 | `내 주장과 근거를 논리적 흐름으로 구성해주세요. 각 슬라이드가 하나의 논점을 담도록 하고, 마지막에 결론을 명확히 제시해주세요.` |
| 📊 데이터/리서치 | `핵심 수치와 통계를 강조하고, 가능한 경우 표로 정리해주세요. 출처를 각 슬라이드 하단에 작은 글씨로 표기해주세요.` |
