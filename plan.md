# 애니메이션 레이아웃 수정 및 프레젠테이션 모드 단일화 계획

본 계획은 애니메이션 적용 시의 레이아웃 깨짐 현상을 수정하고, 프레젠테이션 환경을 OS 전체화면으로 일원화하기 위한 절차를 담고 있습니다.

## 1. 접근 방식
1.  **레이아웃 안정화**: `editor.js`에서 HTML을 가공할 때 서식이 깨지지 않도록 DOM 조작 방식을 개선하고, 매크로 제거 시 공백 처리를 정교화합니다.
2.  **전체화면 일원화**: 프레젠테이션 시작 시 OS 전체화면 진입을 강제하고, 전체화면 해제 시 프레젠테이션이 즉시 종료되도록 연동합니다.

---

## 2. 세부 구현 단계

### 단계 1: 애니메이션 레이아웃 수정 (`src/editor.js`)
*   `processPart` 내에서 `temp.innerHTML` 대신 주석 노드를 직접 제거하고 결과물을 직렬화하는 방식 점검.
*   `.fragment` 스타일이 기존 레이아웃을 해치지 않는지 확인 (예: 불필요한 개행 발생 방지).

### 단계 2: 프레젠테이션 모드 일원화 (`src/presenter.js`)
*   `fullscreenchange` 이벤트 리스너 추가: 사용자가 브라우저 UI나 Esc로 전체화면을 나가면 `this.exit()` 호출.
*   시작 시 전체화면 진입 실패 시 프레젠테이션 중단 처리 (선택 사항) 또는 명확한 안내.

### 단계 3: UI 정리 (`src/presenter.js`, `src/styles/presenter.css`)
*   네비게이션 바에서 불필요한 전체화면 토글 버튼 제거 (이미 강제되므로).
