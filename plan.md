# 스마트 저장 및 강제 저장 기능 구현 계획

불필요한 버전 생성을 방지하고, 명시적인 강제 저장(`Ctrl+S`) 기능과 알림(Toast)을 추가합니다.

## 1. 접근 방식
1.  **데이터 저장 로직 분리**: 실시간 자동 저장(문서 최신화)과 명시적 스냅샷 저장(버전 관리)을 분리합니다.
2.  **단축키 고도화**: 브라우저 기본 동작을 완벽히 차단한 `Ctrl+S`, `Cmd+S` 핸들러를 구현합니다.
3.  **사용자 피드백 강화**: 저장 시 사용자에게 성공 여부를 알려주는 토스트 UI를 도입합니다.

---

## 2. 세부 구현 단계

### 단계 1: 데이터 저장 API 리팩토링
*   **파일**: `src/storage.js`
*   **내용**:
    *   `saveDocument`: 문서 정보만 업데이트하도록 수정.
    *   `saveHistorySnapshot`: 새로운 버전 기록을 생성하는 전용 함수 추가.

### 단계 2: 강제 저장 단축키 핸들러 추가
*   **파일**: `src/main.js`
*   **내용**:
    *   `keydown` 리스너 추가 (`Ctrl+S`, `Cmd+S`).
    *   `e.preventDefault()` 호출 및 `saveContent`, `saveHistorySnapshot` 연계 호출.

### 단계 3: 토스트 UI 구현 및 연동
*   **파일**: `index.html`, `src/styles/editor.css`, `src/main.js`
*   **내용**:
    *   `editor.css`: 토스트 레이아웃 및 애니메이션 스타일링.
    *   `main.js`: `showToast(message)` 유틸리티 함수 구현 및 저장 성공 시 호출.

---

## 3. 고려 사항
*   **중복 저장 방지**: 자동 저장과 강제 저장이 동시에 발생할 경우를 대비해 디바운스 및 상태 확인 로직 유지.
*   **브라우저 호환성**: Mac(`Cmd`)과 Windows/Linux(`Ctrl`) 모두에서 정상 작동하도록 체크.
