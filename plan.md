# 버전 복구 UI 고도화 및 버그 수정 계획

버전 기록 확인 시 드롭다운이 닫히는 버그를 수정하고, 사용자가 요청한 "프리미엄 버전 복구 모달"을 구현합니다.

## 1. 접근 방식
1.  **버그 수정**: `showVersionHistory` 호출 시 드롭다운이 닫히지 않도록 이벤트 전파를 제어하고, `prompt` 대신 커스텀 모달을 사용합니다.
2.  **모달 UI 설계**: 좌측 리스트와 우측 미리보기가 결합된 2컬럼 레이아웃의 모달 창을 구현합니다.
3.  **미리보기 강화**: 버전을 선택할 때마다 우측 영역에 해당 버전의 내용을 즉각 렌더링하여 확인할 수 있게 합니다.

---

## 2. 세부 구현 단계

### 단계 1: 모달 HTML 구조 추가
*   **파일**: `index.html`
*   **내용**: `#history-modal` ID를 가진 모달 컨테이너와 내부 구조(리스트 영역, 미리보기 영역, 버튼군) 추가.

### 단계 2: 모달 스타일링 (Premium CSS)
*   **파일**: `src/styles/editor.css`
*   **내용**: 
    *   배경 블러(Backdrop-filter), 둥근 모서리, 부드러운 트랜지션 적용.
    *   버전 리스트 아이템의 호버 및 활성화 상태 스타일 정의.
    *   미리보기 영역의 스크롤 및 텍스트 스타일 정의.

### 단계 3: 모달 연동 로직 구현 (JS)
*   **파일**: `src/main.js`
*   **내용**:
    *   `showVersionHistory(docId)` 함수 개편: 모달을 열고 해당 문서의 히스토리를 불러와 좌측에 렌더링.
    *   버전 클릭 이벤트: 우측 미리보기 영역 갱신.
    *   복원 버튼 이벤트: 기존 복원 로직 실행 및 모달 닫기.
    *   닫기/외부 클릭 이벤트: 모달 종료.

---

## 3. 고려 사항
*   **성능**: 히스토리가 많을 경우를 대비해 리스트 영역에만 스크롤 적용.
*   **데이터 보호**: 복원 전 현재 상태를 새로운 버전으로 자동 저장하는 기존 안전장치 유지.
