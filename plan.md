# 에디터 고도화 및 설정 시스템 구현 계획 (V2)

본 계획은 검색/치환 기능과 더불어 사용자 맞춤형 설정을 관리할 수 있는 시스템을 구축하기 위한 상세 절차를 담고 있습니다.

## 1. 접근 방식

1.  **검색 시스템 구현**: `Cmd+F` 기반의 검색/치환 UI 및 로직 추가.
2.  **통합 설정 사이드바 구축**:
    *   기존 툴바의 **테마 선택** 및 **레이아웃 버튼**을 사이드바 내로 이동.
    *   줄 간격(Slider), 슬라이드 번호 표시(Toggle) 기능 추가.
    *   우측에서 나타나는 세련된 Glassmorphism 사이드바 구현.
3.  **레이아웃 보강**: 슬라이드 우측 하단에 번호를 배치하고, 줄 간격을 유동적으로 변경할 수 있도록 CSS 변수화.
4.  **데이터 동기화**: 모든 개인화 설정을 IndexedDB에 영구 저장.

---

## 2. 세부 구현 단계

### 단계 1: UI 레이아웃 재구성
*   **파일**: `index.html`
*   **내용**:
    *   툴바에서 테마/레이아웃 요소를 제거하고 설정 아이콘(⚙️) 추가.
    *   우측 설정 사이드바(`#settings-sidebar`) 구조 설계.
    *   에디터 상단 검색바(`#search-bar`) 구조 추가.

### 단계 2: 스타일링 및 디자인 (Premium Aesthetics)
*   **파일**: `src/styles/editor.css`
*   **내용**:
    *   사이드바 및 검색바에 Glassmorphism 효과와 애니메이션 적용.
    *   슬라이드 우측 하단 번호 라벨 스타일링.
    *   줄 간격 조절을 위한 `--slide-line-height` 및 번호 표시 제어 CSS.

### 단계 3: 설정 및 데이터 로직 고도화
*   **파일**: `src/storage.js`, `src/main.js`
*   **내용**:
    *   `storage.js`: 문서 데이터 구조에 `settings` 객체 추가 (lineHeight, showNumber 등).
    *   `main.js`: 사이드바 토글 및 설정 값 실시간 반영/DB 저장 로직 구현.

### 단계 4: 검색/치환 및 강제 개행 기능 구현
*   **파일**: `src/editor.js`, `src/main.js`
*   **내용**:
    *   `Editor` 클래스 내 검색 엔진 로직 구현.
    *   강제 개행 매크로 삽입 기능 구현.
