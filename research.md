# Research: Markdown to Slide 웹앱

## 1. 요구사항 분석

### 사용자 목적
- 유튜브 영상용 슬라이드 제작
- 콘텐츠 유형: 책 내용 정리, 코딩/기술 관련, 생각 공유
- **미니멀한 시각화** 중시
- 웹에서 바로 슬라이드쇼 시작
- 전체화면 모드 지원

### 핵심 기능
1. Markdown 텍스트 → 슬라이드 변환
2. 코드 구문 강조 (코딩/기술 콘텐츠)
3. 슬라이드 네비게이션 (키보드/클릭)
4. 전체화면(Fullscreen API) 지원
5. 실시간 프리뷰
6. 유튜브 영상에 적합한 미니멀 디자인 (16:9 비율)

---

## 2. 기존 도구 분석

| 도구 | 장점 | 단점 |
|------|------|------|
| **Marp** | 미니멀 테마, VS Code 통합 | 웹앱이 아님, 정적 export 중심 |
| **Reveal.js** | 풍부한 기능, 웹 네이티브 | 과도한 기능, 무거움, 커스텀 어려움 |
| **Slidev** | 모던, Vue 기반, 개발자 친화적 | Vue 의존성, 복잡한 셋업 |
| **Remark.js** | 가벼움, Markdown 중심 | 오래된 프로젝트, 제한적 커스텀 |

### 결론
기존 도구들은 사용자의 요구(미니멀, 웹앱, 유튜브용)에 완벽히 맞지 않음.
**Vanilla JS + 경량 라이브러리(CDN)로 커스텀 제작**이 최적.

---

## 3. 기술 스택 선정

### 프론트엔드 (Single Page App, No Framework)
- **HTML5 + Vanilla CSS + Vanilla JS**
- Vite 사용하여 개발 서버 및 빌드 지원

### 핵심 라이브러리 (CDN)
| 라이브러리 | 용도 | 이유 |
|-----------|------|------|
| **marked.js** | Markdown → HTML 변환 | 빠르고 가벼움, 가장 널리 사용 |
| **highlight.js** | 코드 구문 강조 | 다양한 언어/테마, 코딩 콘텐츠에 필수 |

### 브라우저 API
- **Fullscreen API** (`element.requestFullscreen()`) - 전체화면 지원
- **Keyboard Events** - 슬라이드 네비게이션

---

## 4. 슬라이드 구분 방식

Markdown 내 `---` (수평선)을 슬라이드 구분자로 사용:
```markdown
# 제목 슬라이드
부제목

---

## 두 번째 슬라이드
- 항목 1
- 항목 2

---

## 코드 슬라이드
```python
print("Hello World")
```
```

`marked.js`는 `---`를 `<hr>` 태그로 변환하므로, `<hr>`을 기준으로 split하면 자연스럽게 슬라이드 분리 가능.

## 5. 레이아웃 구조

### 에디터 모드 (편집 화면)
```
+------------------------------------------+
|            상단 툴바                       |
| [▶ 슬라이드쇼] [전체화면] [테마 선택]       |
+-------------------+----------------------+
|                   |                      |
|   Markdown        |   실시간             |
|   에디터          |   프리뷰             |
|   (좌측 패널)      |   (우측 패널)         |
|                   |                      |
+-------------------+----------------------+
|          하단 상태바 (슬라이드 수 등)       |
+------------------------------------------+
```

### 프레젠테이션 모드 (전체화면)
```
+------------------------------------------+
|                                          |
|                                          |
|         슬라이드 콘텐츠                    |
|         (16:9, 중앙 배치)                 |
|                                          |
|                                          |
+------------------------------------------+
|  ◀ 이전  |  3 / 10  |  다음 ▶  | [ESC]   |
+------------------------------------------+
```

## 6. 디자인 방향

### 미니멀 디자인 원칙
- **다크 모드 기본**: 유튜브 영상에 적합 (눈 편함, 코드 가독성)
- **넉넉한 여백**: 슬라이드에 정보 과밀 방지
- **제한된 색상 팔레트**: 모노톤 + 1 액센트 컬러
- **모던 타이포그래피**: 한글 지원 폰트 (Pretendard 또는 Noto Sans KR) + 코드용 모노스페이스 (JetBrains Mono)
- **부드러운 트랜지션**: 슬라이드 전환 시 fade 효과

### 유튜브 최적화
- 16:9 비율 고정
- 큰 폰트 사이즈 (모바일 시청 대비)
- 높은 대비 (작은 화면에서도 가독성 확보)

## 7. 참고: Fullscreen API 사용법

```javascript
// 전체화면 진입
const elem = document.documentElement;
if (elem.requestFullscreen) {
  elem.requestFullscreen();
}

// 전체화면 나가기
if (document.exitFullscreen) {
  document.exitFullscreen();
}

// 전체화면 상태 감지
document.addEventListener('fullscreenchange', () => {
  const isFullscreen = !!document.fullscreenElement;
});
```

## 8. 로컬 스토리지 활용

사용자가 작성한 Markdown을 `localStorage`에 자동 저장하여 새로고침 시에도 유지.
추가로 `.md` 파일 내보내기/가져오기 기능도 제공.
